# Story 8.2: Barra lateral de projetos + escopo do canvas

## Status

InReview

## Story

**As a** desenvolvedor multi-projeto,
**I want** uma barra lateral com todos os projetos cadastrados e alternar entre eles,
**so that** eu veja só os terminais/tarefas/workspaces do projeto em que estou focado.

## Acceptance Criteria

1. Barra lateral lista todos os projetos (nome + cor) com o ativo destacado; clicar troca o projeto ativo.
2. Trocar de projeto escopa o canvas, a sessão master, o TasksPanel e o LifecycleBoard para mostrar só sessões/tarefas/workspaces do projeto ativo (mesmo princípio de filtro sem desmontar tiles já usado no workspace da 3.6 — trocar de projeto NÃO mata terminais de outros projetos).
3. Workspaces (3.6) do projeto ativo continuam funcionando exatamente como hoje (seletor de workspace já existente, agora escopado ao projeto).
4. Criar um novo projeto oferece um seletor de pasta (diálogo nativo do SO) para o `rootPath`.

## Dev Notes

- Escopo do AC2 foi além do que o texto sugere à primeira leitura: "sessões/tarefas" exigiu adicionar `projectId` à entidade Task também, não só à sessão — sem isso o TasksPanel/LifecycleBoard não teriam como filtrar. Decisão tomada durante a implementação, documentada aqui em vez de nos Dev Notes originais (mesmo padrão de desvio-documentado da 7.2).
- `ProjectSidebar` fica À ESQUERDA da `Sidebar` de sessões — dois níveis de navegação (projeto → depois workspace/terminal dentro dele), nunca fundidos num componente só, preservando a separação conceitual "projeto ≠ workspace" da decisão de design do épico.
- Cor de projeto: paleta cíclica fixa (6 cores) na criação — nenhum seletor de cor visual nesta story (a AC4 só pede seletor de PASTA); trocar cor depois de criado é possível via `project.update` (já exposto desde a 8.1) mas sem UI dedicada ainda — se necessário, cabe numa story de polish futura.
- Timeline (3.3) foi deliberadamente DEIXADA FORA do escopo por projeto — é uma trilha de auditoria/debug global, não uma superfície de gestão de trabalho; a AC2 lista explicitamente canvas/master/TasksPanel/LifecycleBoard, não Timeline.
