# Story 13.4: Catálogo de agentes

## Status

Done

## Story

**As a** fundador escolhendo qual agente rodar,
**I want** um painel-catálogo listando todos os adapters com descrição, cor, comando e se estão instalados,
**so that** eu saiba de relance o que posso spawnar nesta máquina sem tentar e ver falhar.

## Acceptance Criteria

1. Painel "Agentes" (nova entrada de navegação) lista TODOS os adapters registrados com: nome de exibição, descrição curta, cor de identidade (12.4), comando de spawn e argumento default quando houver (Ollama) (FR45).
2. Disponibilidade real no PATH verificada no MAIN (lookup de executável — nunca no renderer): badge instalado / não encontrado por adapter.
3. Ação "novo terminal com este adapter" direto do catálogo (reusa o fluxo de criação existente, incluindo campo de modelo pro Ollama).
4. A tabela de metadados (descrição/comando por adapter) vive junto de `adapter-colors.ts` no pacote UI — fonte única, sem strings duplicadas no App.tsx.

## Dev Notes

- NFR7 impediu importar `DEFAULT_COMMAND` dos adapters na UI — a tabela é um ESPELHO deliberado, e o teste de paridade com `ADAPTER_COLORS` prende as duas fontes de identidade uma à outra (se um adapter novo ganhar cor sem catálogo, o teste quebra).
- Checagem no PATH sem processo: `where.exe` custaria um spawn por adapter por visita à view; o scan manual com `fs.stat` custa ~8 stats. `PATHEXT` não precisa ser resolvido porque os comandos do catálogo JÁ têm extensão explícita (lição da 12.4/12.6 com CreateProcess).
- `shell`/`cmd` nunca ficam desabilitados: `powershell.exe`/`cmd.exe` vivem em `System32` (sempre no PATH) — e mesmo que a checagem falhe por ambiente exótico, bloquear o shell básico seria pior que deixar tentar.
