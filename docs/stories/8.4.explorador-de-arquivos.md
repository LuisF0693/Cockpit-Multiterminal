# Story 8.4: Explorador de arquivos do projeto ativo

## Status

InReview

## Story

**As a** desenvolvedor multi-projeto,
**I want** uma árvore navegável dos arquivos/pastas do projeto ativo com preview de leitura,
**so that** eu tenha visibilidade do que os agentes estão mexendo sem sair do Cockpit.

## Acceptance Criteria

1. Painel lateral (paralelo ao canvas) exibe a árvore de diretórios do `rootPath` do projeto ativo, expansível/colapsável por pasta.
2. Clicar num arquivo de texto mostra seu conteúdo em modo leitura (sem edição — fora de escopo desta story).
3. Árvore respeita `.gitignore` do projeto quando presente (não lista `node_modules`, `.git`, etc. — ruído demais sem isso).
4. Leitura de diretório/arquivo acontece no Main (Node `fs`), nunca no renderer — mesmo princípio de fronteira de processo já usado em todo o app (contextIsolation, Zod na borda).

## Dev Notes

- Semântica de `.gitignore` é SIMPLIFICADA deliberadamente (documentado no próprio código) — cobre os casos comuns (`node_modules`, `dist`, `*.log`, `.env`, pasta com `/`) sem tentar 100% de compatibilidade com a spec completa (ex.: `**` no MEIO de um padrão como `a/**/b` funciona via split, mas casos mais exóticos podem divergir). Suficiente para a AC3 ("respeita .gitignore... não lista node_modules") sem trazer uma dependência nova (`ignore` npm) para um filtro de UI.
- Só o `.gitignore` da RAIZ do projeto é lido — `.gitignore` aninhados em subpastas não são combinados. Escopo aceitável para v1 (a maioria dos projetos reais tem um `.gitignore` só na raiz); ampliar para múltiplos arquivos fica para uma story de polish futura, se necessário.
- `readFile` valida contenção contra TODOS os projetos cadastrados (não só o ativo) — permite abrir um arquivo de outro projeto se o preview algum dia navegar entre eles; `readDir` é mais restrito (só o projeto pedido/ativo), que é o suficiente para a árvore de UM projeto por vez desta story.
- Gotcha corrigido durante o dev: a implementação original de `isGitignored` usava um caractere-placeholder intermediário na conversão glob→regex que acabou virando um BYTE NUL literal no arquivo fonte (idiossincrasia da ferramenta de escrita, não um bug de lógica) — ESLint (`no-control-regex`) pegou na hora. Reescrito sem placeholder (`split('**')` + `join('.*')` direto) — mais simples e sem esse risco.

### Completion Notes List

- Árvore com expansão lazy (busca filhos só ao expandir) foi uma decisão de implementação não detalhada no AC original mas necessária na prática — ler recursivamente um repositório inteiro de uma vez seria lento e desnecessário para a maioria das navegações.
