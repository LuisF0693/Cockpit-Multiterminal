# Story 10.1: Painel de preview de browser (navegação manual)

## Status

InReview

## Story

**As a** desenvolvedor multi-agente,
**I want** um painel de browser embutido navegável por URL,
**so that** eu possa ver a aplicação que estou desenvolvendo sem sair do Cockpit.

## Acceptance Criteria

1. Novo tipo de tile no canvas: "browser preview" — barra de URL + botão voltar/avançar/recarregar + área de renderização.
2. Preview roda sobre uma instância Playwright (Chromium) gerenciada pelo Main — nunca `<webview>`/`<iframe>` do próprio Electron (precisa ser controlável externamente pelo Playwright para a automação da 10.2).
3. Preview persiste a última URL visitada por tile (mesma garantia de persistência de layout do FR10), mas NÃO precisa persistir estado de navegação profundo (histórico completo) — só a URL atual, reabrir carrega essa URL.
4. Preview é escopado ao projeto ativo (Épico 8) como qualquer outro tile do canvas.

## Dev Notes

- Decisão de design mais importante desta story: a POSIÇÃO/tamanho do tile de browser reusa o `LayoutTile` GENÉRICO já existente desde a 1.4 (chaveado por `id` string, nunca acoplado a `SessionRecord`) — descoberto ao inspecionar `cockpit-store.ts`/`TerminalTile.tsx` e confirmado que `moveTileTo`/`resizeTileTo`/`snapTile`/`bringToFront`/`addTile`/`removeTile` já são id-genéricos. Isso evitou construir um sistema de drag/resize paralelo — só um novo tipo de CONTEÚDO (`BrowserPreviewTile` em vez de `TerminalTile`) sobre a MESMA infraestrutura de canvas.
- "Preview" não é vídeo ao vivo nem interativo por clique direto na imagem — é um SNAPSHOT (screenshot JPEG) atualizado por poll de 1.5s, mesmo princípio de "refresh periódico" já usado em Timeline/SessionReport/ReviewPanel (não há canal de streaming nativo simples via Playwright sem CDP screencast, que seria over-engineering para o escopo desta story — a AC1 pede URL bar+voltar/avançar/recarregar+área de renderização, não interação por clique na própria imagem).
- `ensurePage` idempotente é o que resolve o restore pós-boot (AC3): o `BrowserTile` (URL) sobrevive ao restart via SQLite, mas a `Page` do Playwright NÃO pode sobreviver (processo Chromium é efêmero) — a primeira chamada de `screenshot()` após o boot cria a página sob demanda com a URL persistida, sem exigir lógica de restore explícita separada.
- Build cresceu significativamente (`index.js` do Main: ~30KB → ~6.4MB) por incluir `playwright-core` inteiro no bundle — funcional (smoke:daemon PASS), mas é uma observação de tamanho de build a revisitar numa passada futura de otimização (ex.: marcar `playwright` como external no bundler em vez de inlining), não bloqueante para esta story.
