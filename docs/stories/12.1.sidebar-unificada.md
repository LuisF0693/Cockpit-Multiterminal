# Story 12.1: Barra lateral unificada (projetos + arquivos + preview Markdown)

## Status

InReview

## Story

**As a** desenvolvedor multi-projeto,
**I want** ver meus projetos e os arquivos do projeto ativo na MESMA barra lateral, sempre visível,
**so that** eu não precise trocar de tela pra navegar entre "qual projeto" e "o que tem nele".

## Acceptance Criteria

1. `ProjectSidebar` (8.2) e `FileExplorer` (8.4) passam a viver na MESMA barra lateral persistente (projetos no topo, árvore de arquivos do projeto ativo abaixo) — a view dedicada `'files'` (8.4) é removida do nav, substituída por esta barra sempre visível.
2. Trocar de projeto atualiza a árvore de arquivos exibida automaticamente (reusa `FileExplorer` já existente, só muda ONDE ele é renderizado).
3. Selecionar um arquivo `.md` exibe preview RENDERIZADO (título, listas, código, links) além do modo texto puro já existente (FR35) — outros tipos de arquivo continuam em texto puro.
4. Barra lateral é colapsável (mesmo padrão já usado pela `Sidebar` de sessões, 1.3) para quem quer mais espaço de canvas.

## Dev Notes

- Decisão mais importante desta story: escrever um parser de Markdown PRÓPRIO em vez de adicionar uma dependência (`marked`, `react-markdown`, etc.) — o NFR9 (adicionado no Épico 10) já registrou Playwright como "a única exceção" às dependências mínimas do NFR1; adicionar outra dependência aqui contradiria essa garantia arquitetural recém-formalizada. O parser cobre o caso comum de documentação interna (READMEs, notas) — não é CommonMark completo (sem tabelas, blockquotes, HTML embutido), mas isso não foi pedido.
- `ProjectSidebar` (8.2) e `FileExplorer` (8.4) foram DELETADOS (não deprecados) — `ProjectFilesSidebar` absorve 100% do comportamento de ambos; manter os arquivos antigos sem uso violaria "não deixar código morto".
- Segurança: `renderMarkdownLite` nunca usa `dangerouslySetInnerHTML` — todo o conteúdo passa por tokenização própria e vira elementos React normais (texto/`<strong>`/`<em>`/`<code>`), nunca HTML bruto interpretado pelo browser. Um arquivo `.md` malicioso (ex.: contendo `<script>`) só aparece como TEXTO, nunca executa.
