# Story 16.1: Excluir projeto pela sidebar

## Status

InReview

## Story

**As a** fundador,
**I want** excluir projetos direto na sidebar,
**so that** o cockpit não acumule projetos mortos.

## Acceptance Criteria

1. Botão `×` em cada linha de projeto na sidebar (seção PROJETOS) — oculto quando só resta um projeto (o Main rejeita remover o último, regra da 8.1 preservada).
2. Clicar pede confirmação nativa mostrando o nome do projeto e QUANTOS terminais dele serão fechados (decisão do fundador: fechar sessões junto, sem órfãos).
3. O Main fecha as sessões do projeto (`registry.close`) ANTES de remover — vínculos terminal-a-terminal caem via o listener de `closed` já existente (9.1 AC3); projeto ativo removido troca pro primeiro restante (comportamento do `removeProject` da 8.1).
4. Validação de "último projeto" acontece ANTES de fechar qualquer terminal — falha sem efeito colateral.

## Tasks / Subtasks

- [x] Task 1 — Handler `projectRemove` (Main): valida último projeto, fecha sessões do projeto, remove (AC3, AC4)
- [x] Task 2 — `AppSidebar`: linha de projeto vira flex com botão `×` condicional + prop `onRemoveProject` (AC1)
- [x] Task 3 — App.tsx: `removeProject` com `window.confirm` contando sessões do projeto (AC2)
- [x] Task 4 — `pnpm verify` PASS

## Dev Notes

- Reuso total: `persistence.removeProject` (8.1) já protege o último projeto e troca o ativo; o handler apenas antecipa a validação e fecha as sessões antes.
- Confirmação via `window.confirm` nativo — o app não tem sistema de modais próprio; introduzir um seria escopo novo sem pedido do fundador.
