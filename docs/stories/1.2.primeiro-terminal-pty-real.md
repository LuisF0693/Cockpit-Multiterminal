## Story

**As a** desenvolvedor multi-agente,
**I want** abrir um terminal totalmente interativo dentro do app,
**so that** eu possa executar qualquer CLI (shell, git, node) sem sair do cockpit.

## Acceptance Criteria

1. Terminal renderizado (xterm.js ou equivalente decidido no spike) conectado a um PTY real do sistema.
2. Programas interativos TUI (ex.: `vim`, um CLI agêntico) funcionam corretamente, incluindo cores, cursor e resize.
3. Digitação sem lag perceptível; resize da janela redimensiona o PTY corretamente.
4. Fechar o terminal encerra o processo PTY sem processos órfãos.

## Dev Notes

> Contexto completo dos shards + aprendizados da Story 1.1 — não reler arquitetura.

**Fluxo de dados (docs/architecture/decisao-critica-4-ipc-tipado.md):** dois canais SEPARADOS — controle (Zod, `ipcRenderer.invoke`, baixa frequência) e dados de PTY (MessagePort binário, alta vazão). Misturá-los é o erro que mata a latência de digitação (NFR3 < 16ms). O Main apenas NEGOCIA o canal binário; os dados nunca passam por ele.

**PTY Host (docs/architecture/components.md + high-level):** `utilityProcess` do Electron — isolamento de crash (PTY caiu ≠ UI caiu). Na 1.2 o host usa node-pty diretamente; a abstração de adapter (`packages/adapter-contract`) chega na Story 2.1 — NÃO criar o contrato agora, mas manter o gerenciador de sessões com API limpa (create/write/resize/dispose) para encaixe futuro.

**Gotchas herdados da Story 1.1 (Dev Agent Record):**
1. **electron-builder + pnpm não faz rebuild nativo no Windows** (`pnpm.cjs` não é executável Win32) → `npmRebuild: false` está no `electron-builder.yml`; o rebuild para ABI do Electron é responsabilidade da Task 0 via `@electron/rebuild`. Sem isso, `require('node-pty')` dentro do Electron falha com erro de ABI.
2. **node-pty 1.1.0 emite "AttachConsole failed"** (agente `conpty_console_list_agent`) no kill quando o processo pai não tem console — ruído não-fatal, já documentado em `apps/desktop/spike/conpty-spike-report.json`. Não tratar como erro; suprimir/filtrar no log do host.

**Números de referência do spike (sanity 1 min):** 6 PTYs, RSS ~141MB estável, 0 órfãos pós-kill com verificação `process.kill(pid, 0)`. Reusar esse padrão de verificação de órfãos no `dispose()`.

**Estrutura-alvo:** novo `packages/pty-host` (adicionar ao `pnpm-workspace.yaml` — já coberto pelo glob `packages/*`); exceção de lint para adapters NÃO se aplica ainda (sem adapters). Imports absolutos `@cockpit/*`; adicionar path de `@cockpit/pty-host` no `tsconfig.base.json`.

**Electron hardening:** o renderer continua `sandbox: true` — a MessagePort chega via `postMessage` do preload (portas são transferíveis com sandbox; NÃO desligar sandbox).

**⛔ Pré-condição de início:** gate do spike de 30 min da Story 1.1 aprovado pelo @architect (registro em `docs/architecture/high-level-architecture.md`). Se o spike falhar, esta story é bloqueada e o fallback Tauri reabre o desenho.
