# Story 12.6: Ajustes pós-validação visual do Épico 12 + adapter Ollama

## Status

InReview

## Story

**As a** fundador validando o Épico 12 rodando o app em modo dev,
**I want** que os pontos de atrito encontrados (vínculo por arraste, sidebar fixa, "+" quebrado, fundo de projeto pouco visível, sem zoom) sejam corrigidos/ajustados, e mais um adapter (Ollama),
**so that** a experiência bata com a referência visual (AIOX Cockpit) e os bugs reais achados na validação não fiquem no ar.

## Bugs corrigidos (commit `dcbfb1f`, antes desta story)

1. Adapters `gemini-cli`/`antigravity` spawnavam com nome de comando sem
   extensão (`gemini`/`agy`) — `node-pty` usa `CreateProcess` no Windows, que
   não faz resolução PATHEXT como `where`/cmd.exe. Corrigido para
   `gemini.cmd`/`agy.exe` (mesmo padrão já usado por claude-code/codex/grok).
2. `window.prompt` não é implementado pelo Electron no renderer (retorna
   `null` sempre, sem UI) — quebrava criar/renomear workspace e criar
   projeto silenciosamente (era a causa raiz do "+ não funciona"). Substituído
   por `PromptModal` (novo componente controlado).
3. `project.readDir` enviava `projectId:''` (falha no Zod `min(1)`) quando
   chamado antes do projeto ativo terminar de carregar no boot.

## Acceptance Criteria

1. Vínculo terminal-a-terminal: segurar Alt e arrastar de QUALQUER parte do `TerminalTile` (não só uma alça dedicada) inicia o link — sem Alt, mover/focar continua idêntico (zero regressão).
2. `ProjectFilesSidebar` é redimensionável por arraste (alça na borda direita do painel de arquivos), além do colapsar/expandir já existente.
3. Canvas inteiro ganha um fundo sutil na cor do projeto ativo (além do anel fino por-tile da 12.3) — projeto sem cor cadastrada mantém visual neutro.
4. Canvas tem controle de zoom (botões +/−/reset na toolbar + Ctrl+scroll) — mover/redimensionar tile e o arraste de vínculo continuam 1:1 com o cursor em qualquer nível de zoom.
5. Criar projeto abre o seletor de pasta do Windows diretamente — o nome do projeto vem do nome da pasta escolhida, sem pedir texto antes.
6. NOVO adapter `ollama` — spawna `ollama.exe run <modelo>`, mesmo contrato normalizado (FR3); UI mostra um campo de texto pro modelo (default `llama3`) quando esse adapter está selecionado, já que — diferente dos demais — o argumento não é fixo por adapter, é por sessão.

## Dev Notes

- **Limitação conhecida, não resolvida nesta story:** `args` NÃO sobrevive a restore/relançamento (schema do `SessionRecord` não ganhou coluna nova) — um terminal Ollama relançado após fechar/reabrir o app volta pro modelo default (`llama3`), não o modelo escolhido originalmente. Aceito conscientemente dado o escopo (adicionar persistência exigiria migration de schema, fora do pedido direto do fundador nesta iteração).
- Decisão de design do zoom: o wrapper escalado precisa de `width`/`height` EXPLÍCITOS (extent real dos tiles, não só `100%`) — um `<div>` só com filhos `position:absolute` e sem tamanho próprio colapsa pra 0×0, e a área rolável do `<section>` pai depende do tamanho pós-transform desse wrapper. Sem isso o canvas pararia de rolar corretamente em qualquer zoom ≠ 100%.
- Gesture de vínculo por Alt+drag reusa a MESMA infraestrutura de drag-tracking da 12.2 (`linkDrag` state no App.tsx, `data-tile-id` + `elementFromPoint` pra resolver o alvo) — só a forma de INICIAR o gesto mudou (qualquer ponto do tile + Alt, em vez de uma alça específica); nenhuma mudança no lado do "soltar".
- `OllamaAdapter` é o PRIMEIRO adapter cujo comando de spawn não é fixo — todos os outros (`grok.cmd`, `claude.cmd` etc.) sempre chamam com args vazios. Optou-se por adicionar `SpawnConfig.args` como campo genérico opcional (em vez de um mecanismo Ollama-específico) porque é reutilizável por qualquer adapter futuro com a mesma necessidade — decisão consistente com o contrato de adapter ser "LEI" (Constitution, Artigo I) e servir a TODOS os providers igualmente.
- Hermes (pedido junto de Ollama) foi DELIBERADAMENTE deixado de fora — o fundador não confirmou o comando real de CLI por trás do agente "Hermes" visto na referência (Artigo IV, mesmo critério já usado com Antigravity antes de confirmar `agy`).
