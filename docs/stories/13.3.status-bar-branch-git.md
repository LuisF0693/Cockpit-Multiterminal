# Story 13.3: Status bar global + branch git do projeto

## Status

Done

## Story

**As a** fundador operando vários projetos,
**I want** um rodapé persistente com projeto ativo, branch git, status do daemon, sessões ativas e decisões pendentes,
**so that** o estado vital do sistema esteja sempre visível de relance, como num IDE.

## Acceptance Criteria

1. Status bar persistente no rodapé do app, visível em TODAS as views (canvas e master), com: nome+cor do projeto ativo, branch git, indicador do daemon (reusa o estado da 6.4), contagem de sessões ativas e contagem de decisões pendentes (reusa a fila unificada da 5.3) (FR43).
2. Branch git lida no MAIN (novo canal IPC `project.gitBranch`, leitura de `.git/HEAD` — nunca no renderer), do `rootPath` do projeto ativo; projeto sem repositório git mostra estado neutro (`⎇ —`, sem erro) (FR44).
3. Branch atualiza ao trocar de projeto e em poll de 5s (mesmo padrão do transcript do ReviewPanel) — sem processo git algum (leitura direta de arquivo).
4. Clicar na contagem de decisões pendentes navega para a fila (master) — mesmo atalho do badge do header.

## Dev Notes

- Decisão: ler `.git/HEAD` direto em vez de spawnar `git rev-parse` — zero dependência de PATH, zero processo por poll, e worktrees/submódulos cobertos via `parseGitdirPointer`. Trade-off aceito: estados exóticos (rebase no meio, packed refs para o HEAD — não acontece com HEAD) não são cobertos; qualquer conteúdo inesperado degrada pra `null` (neutro), nunca erro.
- O badge do daemon no HEADER continua só aparecendo quando o vínculo não está saudável (6.4); a status bar mostra o estado SEMPRE (verde quando ok) — papéis diferentes: alarme vs. telemetria.
