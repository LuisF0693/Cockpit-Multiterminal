# Story 13.2: Toolbar do canvas

## Status

Done

## Story

**As a** desenvolvedor multi-agente,
**I want** uma toolbar compacta no canvas com as ações principais (novo terminal, novo preview, zoom, minimapa, overlay de vínculos),
**so that** eu não precise caçar controles espalhados pelo header e cantos da tela.

## Contexto

Segunda story do Épico 13 (FR42). Hoje os controles de zoom (12.6) vivem no
header global (só aparecem com o canvas ativo), o minimapa (12.5) tem só um
colapso interno no próprio widget, e o overlay de vínculos (9.3) não pode ser
ocultado. A referência visual (AIOX Cockpit) tem uma toolbar flutuante no
canvas com ícones. Nasce JÁ sobre os tokens da 13.1.

## Acceptance Criteria

1. Toolbar fixa no canvas (flutuante, não rola com o conteúdo) com ícones + tooltips para: novo terminal (adapter ativo), novo preview de browser, zoom in / out / reset (reusa o estado da 12.6), alternar minimapa (12.5), alternar overlay de vínculos (9.3) (FR42).
2. Os controles de zoom hoje no header são MOVIDOS para a toolbar (sem duplicação de UI de zoom).
3. Alternar o overlay de vínculos esconde/mostra as linhas SVG sem destruir os vínculos (só visual, entidade intocada) — a linha de preview de um arraste em curso continua aparecendo mesmo com o overlay oculto (feedback do gesto não pode sumir).
4. Toolbar usa os tokens da 13.1 e não sobrepõe tiles de forma quebrada (fica ACIMA dos tiles por z-index, fora do wrapper escalado — não encolhe/cresce com o zoom, mesmo padrão do minimapa).

## Dev Notes

- Toolbar fica FORA do wrapper `transform:scale()` (gotcha da 12.6: tudo dentro dele escala com o zoom) e com `position: sticky/absolute` no canto superior do `<section>` rolável — como o section rola, `position: absolute` rolaria junto; usar wrapper `position: sticky` no topo (top:0) com altura 0 pra não empurrar conteúdo, ou `position: fixed` não serve (sai do section). Solução: o minimapa da 12.5 já resolveu isso com `position: absolute` + `right/bottom` — MAS ele fica parado porque é re-renderizado… conferir no código: o minimapa usa absolute dentro do section e NÃO rola junto? `position:absolute` ancora no ancestral posicionado (o section tem `position:relative`), então ELE ROLA com o conteúdo… na prática o canvas raramente rola (extent 2400×1400 vs viewport), e ninguém reportou o minimapa "fugindo". Para a toolbar, usar `position: sticky; top: 8px` num wrapper de altura 0 — imune ao scroll nos dois eixos ficaria complexo; aceitar o MESMO trade-off do minimapa (absolute) mantém consistência. Documentar.
- Novo terminal/browser continuam TAMBÉM no header (funcionam na view master); AC2 só proíbe duplicação do ZOOM.
