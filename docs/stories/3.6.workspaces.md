# Story 3.6: Workspaces (multi-projetos)

## Status

InReview

## Story

**As a** desenvolvedor multi-agente,
**I want** agrupar terminais por workspace/projeto com troca rápida,
**so that** eu opere 4+ projetos simultâneos sem misturar contextos.

## Acceptance Criteria

1. Sessão pertence a um workspace (default: "Geral"); criar/renomear workspaces.
2. Canvas filtra por workspace ativo; sidebar agrupa por workspace.
3. Workspace persiste (state store) e restaura com as sessões.

## Dev Notes

- Workspace é NOME (não id): rename propaga via UPDATE + eventos `renamed`; rename para nome existente = merge (aceitável no MVP).
- `setActive` para workspace inexistente cai para `Geral`; `Geral` é indelével (sempre presente na união).
- Master dashboard NÃO filtra: o conductor governa tudo; filtro é do canvas/sidebar (AC2).
- Migração v3 segue o padrão v2: ALTER guardado por pragma_table_info, instalações novas nascem com a coluna via ALTER idempotente.
