## Dev Notes

**Notify do Codex:** o CLI suporta `notify = ["programa", args...]` no config.toml, disparado em eventos como agent-turn-complete, e aceita override por CLI `-c chave=valor` (TOML). O programa recebe um payload JSON como argumento extra — por isso o parser de linha usa só o primeiro token. Strings TOML LITERAIS (aspas simples) evitam escaping de `\` em paths Windows.

**working sem hook nativo:** o Codex não notifica "prompt submetido" — heurística: qualquer `write()` contendo `\r` marca working (o usuário enviou algo). Dedupe garante que digitação não spama eventos.

**waiting-input:** prompts de aprovação do Codex serão detectáveis por parsing de saída — SEM fixtures reais, fica registrado como debt (mesma política da 2.2).

**Reuso:** padrão ClaudeCodeAdapter (temp dir por sessão, watcher, degradação implícita — sem notify a sessão vive como process-only).
