## Story

**As a** fundador,
**I want** navegar o canvas arrastando o fundo, com todos os projetos visíveis em zonas coloridas,
**so that** eu enxergue e organize a topologia inteira dos meus projetos de uma vez, como no mockup.

## Acceptance Criteria

1. Mundo transformado (`translate(pan) scale(zoom)`, origin 0 0): arrastar o FUNDO faz pan; wheel dá zoom [0.35, 1.6] (sobre um tile, wheel puro pertence ao scrollback do xterm — zoom ali exige Ctrl) (FR49).
2. Terminais de TODOS os projetos visíveis; cada projeto com terminais ganha ZONA (retângulo `radius.xl`, borda `{cor}44`, fundo `{cor}12`, etiqueta pill com dot+nome+contagem) do bounding box + padding 24/label 30 (mock 631-645) (FR49).
3. Projeto ativo = destaque (zona `66`/`1A`) e escopo de criação — trocar de projeto NÃO esconde os demais; workspace (3.6) continua filtrando.
4. Drag de tile/resize/vínculo continuam 1:1 (deltas/zoom — pan não afeta deltas); `pointerToCanvasCoords` vira pan-aware; minimapa deriva o viewport de pan/zoom e clicar CENTRALIZA o pan no tile.
5. Tiles NUNCA desmontam (gotcha xterm 1.3) — pan/zoom/zonas são transformação visual.
