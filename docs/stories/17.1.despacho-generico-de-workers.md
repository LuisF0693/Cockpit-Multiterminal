# Story 17.1: Despacho genérico de workers por agentes

## Status

Done

## Story

**As a** chefe de departamento ou agente orquestrador,
**I want** despachar um worker para o Cockpit informando o agente e a tarefa,
**so that** o sistema escolha um adapter adequado, abra outro terminal no projeto e inicie o trabalho sem troca manual de IA no terminal.

## Acceptance Criteria

1. Uma CLI genérica aceita `--agent`, `--task`, `--cwd` e um `--adapter` opcional; ela não depende do AIOX Master.
2. Sem adapter explícito, uma política pura retorna uma ordem determinística de adapters para desenvolvimento, revisão/planejamento, pesquisa e marketing/conteúdo, com fallback.
3. O dispatcher cria uma sessão externa no daemon com nome do agente, adapter escolhido e instrução inicial; se o adapter falhar ao iniciar, tenta o próximo candidato.
4. A sessão externa é adotada pelo Cockpit com o nome do agente, preservando o projeto inferido pelo `cwd`.
5. Os agentes do projeto recebem documentação de como usar o comando, incluindo o override explícito de adapter.
6. Testes cobrem a política de seleção e a propagação dos metadados de sessão externa.
7. A escolha de modelo é do CHEFE que despacha, caso a caso (decisão do fundador, 2026-07-17): a CLI oferece `--recommend` (consulta a política sem despachar) e a documentação instrui o chefe a justificar a escolha com `--adapter` explícito — perguntando ao usuário quando ambíguo. A política determinística fica como fallback para quando a escolha foi delegada.

## Dev Notes

- O daemon já aceita clientes externos via named pipe e o Main já adota essas sessões em polling. A story estende os metadados enviados pelo protocolo para que a adoção preserve a identidade do agente. [Source: `packages/pty-host/src/daemon-client.ts`, `apps/desktop/src/main/session-ipc.ts`]
- A decisão de adapter fica no core como função pura; o processo CLI executa apenas I/O de daemon. [Source: `packages/core/src/terminal-link-routing.ts`]
- O input inicial deve ser passado na criação da sessão e entregue pelo adapter, evitando simular interação no renderer. [Source: `packages/adapter-contract/src/index.ts`]
