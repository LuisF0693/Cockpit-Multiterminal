## Story

**As a** desenvolvedor multi-agente,
**I want** abrir uma aplicação desktop "Meu Cockpit" instalável na minha máquina Windows,
**so that** eu tenha a fundação sobre a qual todo o cockpit será construído.

## Acceptance Criteria

1. Monorepo criado com `packages/` (core, ui, app no mínimo), TypeScript, lint e typecheck configurados e passando.
2. App desktop abre uma janela com tela inicial (canary page) exibindo nome e versão.
3. Resultado do spike ConPTY documentado em `docs/architecture/` com a decisão de shell (Electron/Tauri) e biblioteca PTY registrada pelo @architect.
4. `npm run dev`, `npm run build`, `npm test`, `npm run lint` funcionais na raiz.
5. CI local mínimo (script de verificação) executa lint + typecheck + testes.

**Spike — por que existe (docs/architecture/high-level-architecture.md):** ConPTY no Windows 10 com ≥6 PTYs é o risco nº 1 do projeto (NFR3). A decisão Electron+node-pty foi tomada com gate de validação: se um critério eliminatório falhar, NÃO prosseguir — escalar ao @architect para acionar o fallback Tauri+portable-pty (a estrutura de packages sobrevive à troca). Ambiente de referência: Windows 10 Home 10.0.19045 (máquina do fundador).

**Persistência/adapters:** fora desta story. Nenhum estado precisa sobreviver a restart ainda (Story 1.4).

**Git:** trabalhar em branch local `feature/1.1-scaffold` (criada pelo @sm/@dev); commits locais convencionais (`feat: ... [Story 1.1]`). Push/PR só via @devops (não há remote ainda — `*environment-bootstrap` pendente).
