# Story 11.1: Entidade Learning (registro manual)

## Status

InReview

## Story

**As a** desenvolvedor multi-projeto,
**I want** registrar um aprendizado com texto livre, categoria e o projeto de origem,
**so that** gotchas e decisões que valeram a pena fiquem capturados em vez de esquecidos.

## Acceptance Criteria

1. Um learning tem `id`, `text` (texto livre), `category` (string livre curta — ex.: "gotcha", "decisão", "padrão"), `projectId` de origem (nullable — pode ser registrado sem projeto ativo) e `status` de qualificação (`draft` por padrão).
2. Learning é capturável a partir da sessão master (campo de captura rápida) — sem exigir navegação a uma tela dedicada para o caso comum.
3. Learnings persistem independentemente de reinicialização do app (mesma garantia do FR10) e independentemente da existência do projeto de origem (FR31) — remover o projeto (Épico 8) não remove learnings associados.
4. Nenhuma tabela por-projeto: um único armazenamento global, mesmo banco SQLite já usado pelo resto do app (não é infraestrutura de persistência nova, só uma tabela nova).

## Dev Notes

- `canTransitionLearning`/`VALID_TRANSITIONS` (as 4 transições da 11.2: `draft→reviewed`, `draft→discarded`, `reviewed→reusable`, `reviewed→discarded`) foram construídas JUNTO nesta story porque o `LearningManager.updateStatus()` precisa validar a transição desde o primeiro commit (senão a entidade nasceria sem nenhuma garantia de integridade) — mas a UI de qualificação (fila, botões de decisão) é escopo da 11.2, não desta story.
- Gotcha de teste corrigido durante o dev (mesma classe já documentada neste projeto): o teste original de `list()` ordenando por `createdAt` usava `manager.create()` duas vezes em sequência, colidindo no mesmo milissegundo de wall-clock — reescrito para usar `store.createLearning()` com `createdAt` EXPLÍCITO + `manager.load()`, eliminando a dependência de tempo real.
- Contador "N no banco" na captura rápida (não pedido explicitamente pela AC, mas necessário para a leitura do `learnings` state não ficar "morta" entre esta story e a 11.3, que ainda não existe) — decisão de UX mínima documentada aqui.
