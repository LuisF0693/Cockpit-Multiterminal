# Story 18.3: Vínculo automático respeita "mesmo projeto" como o manual

## Status

Ready

## Story

**As a** mantenedor do Cockpit,
**I want** a regra "vínculo só entre terminais do mesmo projeto" centralizada num único lugar,
**so that** o vínculo manual e o automático (Épico 17) nunca voltem a divergir de comportamento por estarem implementados em cópias separadas.

## Acceptance Criteria

1. Existe uma única função pura (`@cockpit/core`) que decide se dois terminais podem ser vinculados por projeto — usada tanto pelo handler manual (`terminalLinkCreate`) quanto pelo fluxo de adoção automática (`adoptExternalSessions`, Story 17.2).
2. Comportamento observável PRESERVADO: manual continua lançando erro pro chamador; automático continua apenas logando e não criando o vínculo (não pode lançar — é um `setInterval` sem chamador aguardando resposta).
3. `projectId === null` NUNCA conta como "mesmo projeto" em nenhum dos dois caminhos (guarda já corrigida na revisão da PR #13 — esta story move a lógica pro lugar único, não muda o resultado).
4. Testes cobrem a função pura isolada (inclusive o caso dos dois `null`) e os dois call sites continuam verdes.

## Tasks / Subtasks

- [ ] Extrair `isSameProject(a: string | null, b: string | null): boolean` (ou nome similar) em `@cockpit/core` — trata `null` explicitamente como "nunca igual".
- [ ] Trocar a comparação inline em `session-ipc.ts` (handler manual, linha ~542) e no fluxo automático (linha ~767) pela função nova.
- [ ] Testes unitários da função pura.
