# Story 12.3: Fundo colorido por projeto nos terminais

## Status

Done

## Story

**As a** desenvolvedor multi-projeto,
**I want** que os terminais do projeto ativo tenham uma cor de fundo/borda reconhecível,
**so that** eu identifique de relance a qual projeto cada terminal pertence, sem ler o nome.

## Acceptance Criteria

1. `TerminalTile` (e `BrowserPreviewTile`) recebem uma indicação visual (borda ou fundo sutil) na cor do PROJETO ao qual o terminal pertence (FR21) — reusa a cor já cadastrada no projeto, nenhuma paleta nova.
2. Terminais sem projeto (dado pré-Épico-8, raro após o backfill) não mostram cor — visual neutro atual, sem regressão.
3. A cor de projeto é sutil o bastante pra não brigar com os indicadores de status já existentes (pulso de waiting-input, borda de foco) — camadas visuais coexistem sem confusão.
4. Funciona igual em qualquer workspace (3.6) dentro do projeto — a cor é por PROJETO, não por workspace.

- [x] Task 1 — `TerminalTile`/`BrowserPreviewTile` (packages/ui): nova prop opcional `projectColor?: string | null`; quando presente, renderiza um `<div>` overlay `position:absolute inset:0 pointerEvents:none` com `boxShadow: inset 0 0 0 2px {cor}55` (AC1) — camada INDEPENDENTE do `boxShadow`/`animation` já usado pro pulso de waiting-input e pro anel de foco (AC3): nenhuma propriedade CSS é compartilhada, então não há risco de uma animação sobrescrever a outra

- [x] Task 3 — nenhuma mudança de filtro por workspace: a cor é derivada só de `projectId`, nunca de `workspace` — funciona igual em qualquer workspace dentro do projeto (AC4), sem código adicional (já era assim, só reaproveitado)

## Dev Notes

- Cor vem da paleta cíclica FIXA já usada na criação de projetos (Story 8.2, App.tsx `PROJECT_COLORS`, 6 cores hex de 6 dígitos, ex. `#3B82F6`) — nenhuma paleta nova criada nesta story, só reaproveitamento (AC1 explicitamente pede isso).
- Opacidade `55` (hex, ~33%) aplicada só no anel INSET de 2px, não no fundo inteiro do tile — decisão deliberada de "borda" em vez de "fundo" (AC1 permite qualquer um dos dois) porque um fundo tingido em toda a área competiria visualmente com o conteúdo do terminal (texto do xterm) ou com o screenshot do browser preview; um anel fino nas bordas dá a mesma identificação de relance sem essa competição.
- A camada de cor de projeto é um elemento DOM totalmente separado do `<section>` cujo `boxShadow`/`animation` já fazem o pulso de waiting-input e o anel de foco — evita qualquer risco de uma propriedade CSS sobrescrever a outra (o que aconteceria se tudo fosse empacotado num único `boxShadow` compartilhado, já que CSS animations substituem o valor inteiro da propriedade animada).
