# Story 15.4: Central de API (chaves no keychain do SO)

## Status

Done

## Story

**As a** fundador,
**I want** cadastrar providers de LLM (tipo, apelido, base URL, chave, modelo) com a chave protegida pelo keychain do Windows,
**so that** os futuros adapters cloud já encontrem as credenciais prontas — sem nunca gravar chave em texto plano.

## Acceptance Criteria

1. Seção "Central de API" na janela de Configurações (15.3): CHAVES CADASTRADAS (tipo/apelido/baseUrl/modelo — a chave NUNCA é reexibida) + CADASTRAR PROVIDER (Tipo, Nome, Base URL, API Key em campo password, Modelo default opcional), fiel à foto (FR56).
2. Chave criptografada com `safeStorage.encryptString` (DPAPI/keychain) — ciphertext base64 em `app_meta.api_providers`; sem `isEncryptionAvailable()`, cadastro RECUSADO com mensagem (nunca texto plano).
3. Remover apaga o registro (ciphertext junto); NENHUM canal devolve a chave (nem cifrada) ao renderer — não existe decrypt hoje (sem consumidor, decisão consciente do fundador).
4. Documentado no rodapé da própria tela: nenhum adapter consome ainda (preparação; NFR6 preservado pros CLIs).
