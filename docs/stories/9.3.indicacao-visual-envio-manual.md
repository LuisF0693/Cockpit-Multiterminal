# Story 9.3: Indicação visual no canvas + envio manual

## Status

InReview

## Story

**As a** desenvolvedor multi-agente,
**I want** ver os vínculos entre terminais desenhados no canvas e poder disparar manualmente um vínculo `manual`,
**so that** eu entenda a topologia de comando entre agentes de relance.

## Acceptance Criteria

1. Canvas desenha uma linha/indicador conectando os tiles de terminais vinculados (origem→alvo), visível sem precisar abrir um painel separado.
2. Vínculo em modo `manual` expõe um botão "enviar" na sessão master (e/ou no tile) que dispara a instrução sob demanda, reusando o `instructAgent` já existente (3.2).
3. Criar/remover um vínculo é possível a partir do canvas (ex.: arrastar de um tile a outro) OU da sessão master (dropdown), qualquer um dos dois caminhos é aceitável — não é obrigatório ter os dois.
4. Vínculos aparecem também numa lista simples na sessão master (origem → alvo, modo) para quem prefere texto a desenhar linhas.

## Dev Notes

- Optou-se por NÃO implementar criação de vínculo via arrastar-no-canvas (a outra metade do "ou" da AC3) — a mesma superfície do `TerminalTile` já tem drag-to-move; sobrepor um segundo gesto de drag (criar vínculo) arriscaria colisão de interação sem trazer valor adicional, já que o dropdown do master cobre a AC integralmente. Fica documentado como possível extensão futura, não como pendência desta story.
- Overlay SVG fica DENTRO do mesmo `<section>` posicionado `relative` que já contém os tiles absolutos — usa as MESMAS coordenadas `x/y/width/height` do `layout.tiles`, sem sistema de coordenadas paralelo.
- `sendTerminalLink` e `planTerminalLinkRouting` (9.2) compartilham a INTENÇÃO (avaliar o trabalho da origem) mas não compartilham código — a mensagem manual é redigida separadamente no renderer porque o gatilho é humano (não há "status novo" a descrever) e a redação já é curta o bastante para não justificar uma abstração compartilhada.
