# Page Composition Checklist

---

## 1. Layout Grid System

- [ ] Uses `grid-cols-12` (or equivalent 12-column layout strategy)
- [ ] Uses appropriate container widths (`max-w-*`) per page context
- [ ] Applies a valid pattern from `squads/aiox-design/data/page-layout-framework.md`

## 2. Typography Hierarchy

- [ ] Exactly one H1 per page
- [ ] Type scale progression follows modular ratio (1.25)
- [ ] Body text uses readable measure (`max-w-prose` when applicable)
- [ ] Line-height supports readability by role (display/heading/body/caption)

## 3. Spacing Rhythm

- [ ] Spacing tokens are consistent (`space-*`, `gap-*`, `p-*`, `m-*`)
- [ ] Proximity law is respected (intra-group < inter-group spacing)
- [ ] A clear whitespace strategy is used (Generous, Balanced, or Compact)

## 4. Mobile-First Responsive

### 4.1 Responsive Layout

- [ ] Mobile-first class order: base style without prefix, progressive with `sm:`/`md:`/`lg:`
- [ ] Grids scale progressively: `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3` (never fixed multi-column)
- [ ] Responsive gaps: `gap-4 lg:gap-8` (never fixed `gap-8`+)
- [ ] Responsive paddings: `p-4 sm:p-8 lg:p-12` (never fixed `p-12`+)
- [ ] No horizontal overflow at 375px viewport width

### 4.2 Mobile Navigation

- [ ] If using `hidden md:flex` or `hidden lg:flex`, a mobile alternative MUST exist (hamburger/drawer)
- [ ] Mobile menu is keyboard-accessible and closable with Escape

### 4.3 Mobile Typography

- [ ] Headings use minimum 3 breakpoints: `text-2xl sm:text-3xl lg:text-5xl` (never skip from mobile to desktop)
- [ ] Body text: `text-base sm:text-lg` (never fixed `text-lg`)
- [ ] `max-w-md`/`max-w-lg` verified to fit at 375px with side padding (`px-4` = 343px available)

### 4.4 Touch Interaction

- [ ] Touch targets minimum 44x44px on all interactive elements (buttons, links, nav items)
- [ ] Minimum 8px spacing between adjacent tappable elements
- [ ] Button groups use `flex-col sm:flex-row` (stack on mobile, side-by-side on desktop)

### 4.5 Media and Performance

- [ ] Below-fold videos/images use lazy loading (`IntersectionObserver` or `loading="lazy"`)
- [ ] Background videos do not block scroll on slow devices (no FCP/LCP impact)

## 5. CTA Contrast

- [ ] Primary CTA uses high-contrast styling (for example `bg-primary text-primary-foreground`)
- [ ] Primary CTA has strongest visual prominence in conversion zones
- [ ] CTA placement is strategic for page intent and reading flow

## 6. Copy-to-Layout Alignment

- [ ] Copy framework is mapped correctly (AIDA, PAS, or StoryBrand)
- [ ] Visual weight aligns with section intent
- [ ] Eye pattern guidance is applied (Z/F pattern as appropriate)

## 7. Accessibility WCAG AA

- [ ] Text contrast target meets 4.5:1 for normal text
- [ ] Interactive elements expose visible focus styles (`focus-visible`)
- [ ] Semantic landmarks are present (`header`, `nav`, `main`, `footer`)
- [ ] Informative images include meaningful `alt` text

## 8. Anti "Cara de IA"

- [ ] Avoids generic gradient defaults (for example overused purple-pink hero presets)
- [ ] Spacing rhythm shows intentional variation, not uniform repetition
- [ ] Typography shows hierarchy nuance, not one-size-only scaling
- [ ] Composition includes project-specific customization beyond default block assembly
- [ ] No default indigo/blue palette without brand reasoning
- [ ] No placeholder content (lorem ipsum, "Feature 1", placeholder images)
- [ ] Components customized beyond shadcn defaults (variants, colors, sizes applied)

> Full catalog: `squads/aiox-design/data/anti-ai-look-patterns.md` (10 anti-patterns with fixes)

## 9. SEO Metadata

- [ ] Title tag exists and is under 60 characters with primary keyword
- [ ] Meta description exists (120-160 characters) with CTA
- [ ] Exactly one H1 with primary keyword (cross-ref Section 2)
- [ ] Semantic HTML landmarks present (`header`, `main`, `footer`, `nav`, `section`)
- [ ] Images below fold use `loading="lazy"`, all have `alt` text
- [ ] JSON-LD structured data included (for indexable page types)
- [ ] Canonical tag present (for indexable page types)
- [ ] Auth/settings pages have `noindex` meta tag

> Full checklist: `squads/aiox-design/checklists/seo-meta-checklist.md`
> Rules reference: `squads/aiox-design/data/seo-rules.md`

## 10. Component States

- [ ] All data-dependent sections have loading state (Skeleton components)
- [ ] All data-dependent sections have empty state (message + CTA)
- [ ] All data-dependent sections have error state (message + retry)
- [ ] Conditional rendering pattern applied consistently

> Task reference: `squads/aiox-design/tasks/compose-component-states.md`
