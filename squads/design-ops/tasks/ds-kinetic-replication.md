# Kinetic Replication Engine (CSS-only v1)

> Task ID: ds-kinetic-replication
> Owner: `design-chief`
> Version: 1.0.0
> **Execution Type:** `Agent`
> **Dependencies:** depends_on: `[ds-motion-extraction]` . enables: `[ds-visual-regression]` . workflow: `motion`

## Description

Receive the motion map (output of Phase 3 extraction) and the motion primitives
registry, then generate production-ready CSS code that replicates the extracted
animations using the design system's primitive library. This is Phase 4 of the
Kinetic Design System pipeline — the step that closes the loop from "extract"
to "generate code".

**Scope v1 (D3):** CSS-only output. No Framer Motion generation, no GSAP
generation, no AI Vision classification. Matching is deterministic, based on
CSS property signatures (opacity -> fadeIn, transform -> slideIn/scaleIn,
filter -> blur, clip-path -> clipReveal, color/background-color -> colorShift).

## Prerequisites

- Motion map generated by `ds-motion-extraction` (`motion-map.json`)
- Motion primitives registry loaded (`data/motion-primitives-registry.yaml`)
- Motion tokens schema loaded (`data/motion-tokens-schema.yaml`)

## Workflow

### Interactive Elicitation

1. **Target Project Path**
   - Path where generated CSS will be placed (required)
   - e.g., `src/styles/motion/` or `app/styles/`

2. **Output Format**
   - **CSS Modules:** Scoped `.module.css` files per component
   - **Utility Classes:** Global utility classes (`.motion-fadeIn`, `.motion-slideIn`, etc.)
   - **@layer:** CSS `@layer motion-replicated { ... }` block (recommended per D7)

3. **Motion Map Path**
   - Path to `motion-map.json` (default: output from `ds-motion-extraction`)

### Steps

1. **Load Inputs**
   - Read `motion-map.json` (output of Phase 3)
   - Read `data/motion-primitives-registry.yaml` (primitive catalog)
   - Read `data/motion-tokens-schema.yaml` (token references)
   - Validate motion-map structure against `data/motion-map-schema.json` (JSON Schema 2020-12 -- REQUIRED)
   - Check: all 3 inputs loaded AND motion-map has entries -- log "Inputs loaded: {N} motion-map entries, {N} primitives in registry"

2. **Match Animations to Primitives**
   - For each entry in motion-map:
     - Read `css_properties` from the entry (e.g., `["opacity"]`, `["transform"]`, `["filter"]`)
     - Match against primitive `css_properties` in registry:
       - `opacity` only -> `fadeIn` or `fadeOut` (check direction: 0->1 = fadeIn, 1->0 = fadeOut)
       - `transform: translate*` -> `slideIn` or `slideOut` (check direction)
       - `transform: scale*` -> `scaleIn`
       - `transform: rotate*` -> `rotate`
       - `filter: blur*` -> `blur`
       - `clip-path` -> `clipReveal`
       - `color` or `background-color` -> `colorShift`
       - `transform: translate* + scroll-driven` -> `parallax`
     - Multiple matching properties -> classify as composition (match to registry compositions)
     - No match -> mark as `"unclassified"`
   - Record confidence per match:
     - Single property, exact match -> ALTA
     - Multiple properties, partial match -> MEDIA
     - Ambiguous or complex keyframes -> BAIXA
   - Check: classification complete -- log "Matched: {N} ALTA, {N} MEDIA, {N} BAIXA, {N} unclassified"

3. **Generate CSS @keyframes**
   - For each classified animation:
     - If primitive exists in registry, use the `css_pattern` as template
     - Replace hardcoded values with CSS custom property references from tokens schema
     - Generate `@keyframes` rule with unique name: `{primitive}-{element-hash}`
     - All durations reference `var(--motion-*)` tokens (zero hardcoded values)
     - All easings reference `var(--motion-easing-*)` tokens
   - For unclassified animations:
     - Copy raw keyframes from motion-map as-is
     - Wrap in commented block: `/* UNCLASSIFIED: review manually */`
   - Check: keyframes generated -- log "Generated: {N} @keyframes ({N} from primitives, {N} raw/unclassified)"

4. **Generate Utility Classes**
   - For each generated @keyframes:
     - Create utility class: `.motion-{primitive}` with `animation` shorthand
     - Create scroll-driven variant: `.motion-{primitive}--scroll` with `animation-timeline: view()`
     - Create hover variant: `.motion-{primitive}--hover` triggered on `:hover`
   - If output format is `@layer`:
     - Wrap all in `@layer motion-replicated { ... }`
   - If output format is CSS Modules:
     - Generate per-component `.module.css` files
   - Check: utility classes generated -- log "Utility classes: {N} standard, {N} scroll-driven, {N} hover"

5. **Generate Reduced-Motion Fallbacks**
   - For each generated animation:
     - Read `reduced_motion` field from the matched primitive in registry
     - If `"Keep"`: no override needed (opacity-only, color-only are low-motion)
     - If `"Remove"`: `@media (prefers-reduced-motion: reduce) { animation: none; }`
     - If `"Replace with fadeIn"`: substitute with fadeIn at reduced duration (100ms ease-out)
     - Strategy follows D8: "reduced, not removed" — functional animations get reduced durations (100-300ms), decorative animations get removed
   - Generate `@media (prefers-reduced-motion: reduce)` block with all overrides
   - Check: reduced-motion fallbacks generated -- log "A11y: {N} kept, {N} reduced, {N} removed"

6. **Validate Output**
   - Verify zero hardcoded duration/easing values (all reference CSS custom properties)
   - Verify every generated class has a reduced-motion fallback or explicit "Keep" reason
   - Verify all token references resolve against `motion-tokens-schema.yaml`
   - Count unclassified animations (acceptable but flagged for manual review)
   - Generate validation summary in report
   - Check: validation passed -- log "Validation: {N} token refs OK, {N} a11y fallbacks OK, {N} warnings"

7. **Generate Replication Report**
   - Compile findings into `replication-report.md`:
     - Summary: total animations processed, classification breakdown
     - Per-element detail: selector, matched primitive, confidence, token mapping
     - Unclassified section: raw CSS with review recommendations
     - Token coverage: which tokens from schema were used vs unused
     - Reduced-motion audit: strategy per animation
   - Check: report generated -- log "Report: {N} sections, {N} unclassified flagged"

## Output

- `replicated-motion.css` -- All generated @keyframes, utility classes, scroll-driven variants, and reduced-motion fallbacks
- `replication-report.md` -- Human-readable classification and validation summary
- `.state.yaml` updated

## Failure Handling

- **motion-map.json not found:** Halt with error "Run ds-motion-extraction first (Phase 3 must complete before Phase 4)"
- **motion-map.json empty (0 entries):** Report "No animations found in extraction — static page or extraction failure"
- **Primitive not matched (unclassified):** Do NOT halt — mark entry as `"unclassified"`, include raw CSS in output with `/* UNCLASSIFIED */` comment, flag in report for manual review
- **Token reference unresolved:** Warn in report, fall back to inline value with `/* TODO: map to token */` comment
- **motion-primitives-registry.yaml not found:** Halt with error "Registry required — run ds-motion-primitives first"

## Success Criteria

- All motion-map entries processed (classified or explicitly unclassified)
- Generated CSS uses only token references (zero hardcoded durations/easings)
- Every animation has reduced-motion strategy documented
- Unclassified animations preserved as raw CSS with review flags
- Report includes per-element classification with confidence levels
- Output file is valid CSS (parseable, no syntax errors)
